## ✨ Key Features

### Authentication
```
User Login → JWT Token with Role → localStorage
         ↓
    App checks role
         ↓
Redirects to appropriate dashboard
```

### Authorization
```
Frontend: ProtectedRoute checks role → Allow or Redirect
Backend: Middleware validates JWT → Check role → Allow or 403
```

### Notes Feature
```
Create Note → Display → Edit → Delete
All with beautiful UI, timestamps, error handling
```

### Role-Based Access
```
Admin   → /admin     Dashboard (admin-only)
Teacher → /teacher   Panel (teacher-only)
Student → /student   Dashboard + Notes (student-only)
```

### For Testing
```javascript
// Check user role in console:
JSON.parse(localStorage.getItem('user')).role

// Test API access:
fetch('http://localhost:5000/api/admin/stats', {
  headers: { Authorization: `Bearer ${localStorage.getItem('token')}` }
}).then(r => r.json()).then(console.log)
```

### For Debugging
```javascript
// Check localStorage:
localStorage

// Check current user:
console.log(JSON.parse(localStorage.getItem('user')))

// Check token:
console.log(localStorage.getItem('token') ? 'Exists' : 'Missing')
```
